Yates Editorial OS

Quiet luxury software for a blog that can actually ship on GitHub Pages.

The frontend stays static and portable. Authentication, permanent accounts, posts, and realtime rooms are connected through a hosted browser-safe data layer.

Deployment model

Upload these files directly to a GitHub Pages branch or folder. No terminal is required after the files are prepared.

Architecture overview

Static shell. Hosted identity. Browser-owned AI key.

This app is built for your exact constraint: files that can be dragged into GitHub and served by GitHub Pages. The page itself is static, while global users, chat rooms, posts, and account deletion are handled through a hosted service that exposes a public project URL and anon key.

Owner role Only the designated owner can publish blog posts.
Model switching Users can choose supported AI models once a local AI key is present.
Persistence Accounts and chat live remotely, not only in local storage.

Important reality

A static site cannot safely hide an owner-wide AI provider key. This package uses a browser-only personal key field instead. Public data access is controlled by hosted security policies, not by hiding the anon key.
